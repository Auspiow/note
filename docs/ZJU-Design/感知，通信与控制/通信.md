### 通信历史

#### 1G：模拟蜂窝通信的起点（1980年代）

多址方式：FDMA（频分多址）
调制方式：模拟调频（FM）
商用网络：AMPS美国1983 / NMT北欧1981
工作频段：800～900 MHz
传输内容：仅支持语音，无数据传输（模拟信号）
致命缺陷：无加密，通话可被截听

标志性产品：摩托罗拉DynaTAC 8000X（1983）

中国：1987年广东开通首个1G网络，全部进口爱立信设备与摩托罗拉终端——纯粹的使用者

> 频率复用原理
>
> 将覆盖区域划分为正六边形小区，相邻小区使用不同频率组（A/B/C），足够远的小区可以复用同一频率，在有限频谱下实现大面积覆盖。蜂窝架构是此后所有移动通信系统的基础设计框架，"Cell"的概念延续至今。

#### 2G：从模拟到数字的根本性跨越（1990年代）

多址方式：TDMA（时分多址）
核心标准：GSM — 全球最广泛部署
三大创新：
 · SIM卡（用户身份与终端分离）
 · A5/1算法语音加密
 · SMS短信（1992年首条商用）
GPRS（2.5G，按量计费）EDGE（2.75G）：改进了调制方式，速度提升到 384kbps。

CDMA IS-95（Qualcomm高通）：北美/韩国 
原理：所有人都在同一个频率说话，但每个人说不同的语言（特征码）。接收端只要知道你的“语言种子”，就能从嘈杂的背景音中精准提取你的声音。
软切换优势，3G CDMA技术前身。

标志性产品：
诺基亚3310（2000年）：累计销量1.26亿部，"坚不可摧"文化符号
摩托罗拉StarTAC（1996年）：全球首款翻盖手机，重量仅88克开创手机"小型化设计"竞赛
爱立信T28（1999年）：超薄滑盖，引领工业设计审美

中国：华为1996年进入GSM设备市场，深圳华强北供应链崛起，联想/波导推出国产品牌——跟随者萌芽期

#### 3G：三足鼎立的国际标准竞争（2000年代）

WCDMA · CDMA2000 · TD-SCDMA：谁的标准，谁的话语权

| **标准名称** | **背后推手**  | **技术特点**                       | **历史地位**                           |
| ------------ | ------------- | ---------------------------------- | -------------------------------------- |
| WCDMA        | 欧洲/全球主流 | 演进自 GSM，带宽大，技术最成熟     | 全球覆盖最广，中国联通曾持此牌照       |
| CDMA2000     | 美国（高通）  | 演进自 IS-95，升级平滑，语音质量好 | 曾风靡北美和日韩，中国电信曾持此牌照   |
| TD-SCDMA     | 中国          | 时分双工，频谱利用率高             | 中国首个拥有自主知识产权的国际通信标准 |

初代iPhone × App Store：移动互联网经济的开关
2007年初代iPhone以EDGE（2.75G）发布，2008年iPhone 3G接入3G网络，配合App Store同步上线，开启"移动应用经济"时代。

HTC：凭借精良Android硬件设计，3G时代短暂登顶全球智能机销量第一
三星 Samsung：Galaxy系列在3G末期确立"旗舰Android机"品牌定位，为4G爆发蓄势
诺基亚 Nokia：Symbian封闭架构无法适应触控App生态，3G末期被迫退出领导地位
华为/中兴 · TD-SCDMA战略意义：建立完整基站研发体系，培养核心工程团队，为4G全球竞争奠定基础

中国：2009年TD-SCDMA商用，中国从通信标准旁观者首次成为参与者，具有里程碑式历史意义

#### 4G LTE——移动互联网全面爆发（2010年代）

多址方式：OFDMA（正交频分多址）：子载波正交切分，有效抵抗多径衰落
天线技术：MIMO（多入多出）：多天线空间复用，大幅提升频谱效率
峰值下行速率：100Mbps ～ 1Gbps
端到端延迟：10 ～ 50ms
首次实现：VoLTE（语音全IP化）
中国贡献：深度主导TD-LTE国际标准

华为（Huawei）：4G设备全球份额第一，超越爱立信，跻身全球顶级供应商
高通（Qualcomm）：Snapdragon 800系列，三星/LG/索尼等旗舰首选基带芯片
联发科（MediaTek）：中低端市场，支撑发展中国家4G智能机规模普及
苹果（Apple）：iPhone 6引爆4G换机潮，采用英特尔+高通双基带方案

中国：中国移动2013年获全球首批4G牌照，建成最大TD-LTE网络；华为成全球设备商主导者

假设今天只有1G网络（仅语音，无数据），哪些应用无法实现？若升级到2G（GPRS 64kbps），哪些功能重新出现了？升到3G（HSDPA 14Mbps）后呢？

1. 假设只有 1G 网络（仅模拟语音，无数据）

   手机只有一个功能：纯语音通话。

2. 升级到 2G（GPRS/EDGE，约 64kbps）
   纯文字通信：短信（SMS）。
   基础版网页（WAP）：你可以浏览那种没有任何图片、只有蓝色超链接的纯文字新闻网页。
   手机 QQ：你可以和好友聊天了，但发送一张照片可能需要等待 1 分钟甚至更久。
   简单的股票看板：刷新一次数字只需要几 KB 流量，2G 勉强能应付。

3. 升级到 3G（HSDPA，约 14Mbps）
   图片社交：微信朋友圈、微博开始流行。可以随时随地分享自拍，虽然原图上传还是有点慢，但压缩图完全没问题。
   移动支付：扫码支付变得极其可靠。在 3G 环境下，加载一个支付二维码只需一瞬间。
   地图与网约车：实时地图（高德/百度）可以流畅加载，滴滴/Uber的原型开始出现，因为手机可以实时向服务器发送位置坐标了。
   应用商店（App Store）：可以从云端下载软件，而不是靠数据线连接电脑传输。
   音乐流媒体：手机听歌不再需要提前存入 MP3，可以实时在线试听。

#### 5G三大应用场景：ITU-R IMT-2020战略框架

eMBB · uRLLC · mMTC — 面向不同需求，5G提供三种截然不同的能力包

eMBB（增强型移动宽带）：给人用，更快。
uRLLC（超高可靠低时延通信）：给“机器”用的，特别是那些对时间极其敏感的机器，延迟要求低于 1 毫秒，且可靠性达到 99.999%。自动驾驶、远程手术、工业自动化控制。
mMTC（海量机器类通信）：给“万物”用的，要求能同时连接大量设备，并且低功耗。智慧城市（井盖、路灯、水表全联网）、智能工厂。

峰值下行速率：20Gbps（4G: 1Gbps）→ 约20倍提升
用户体验速率：100Mbps ～ 1Gbps  → 保障流媒体/VR稳定体验
频谱效率：提升约3倍 → OFDM NR + Massive MIMO贡献
系统容量：提升约100倍  → 密集组网 + 新频谱协同实现

> 4G下载一部4K电影 ≈ 6分钟  5G下载一部4K电影 ≈ 10秒

延迟与连接密度（uRLLC + mMTC）

端到端延迟：1ms（4G: 30~50ms）→ uRLLC场景的核心生命线指标
可靠性：99.999%（五个九）→ 工业控制和医疗级前提条件
连接密度：100万设备/km² → 4G: 10万/km²，提升10倍
移动速度支持：500km/h → 高铁/航空场景无缝连续覆盖"

**Massive MIMO：5G最核心的天线技术革命**

| 技术                    | 功能说明                                                     |
| ----------------------- | ------------------------------------------------------------ |
| 天线数量的跨代跃升      | 4G MIMO典型配置2~8根天线；5G商用Massive MIMO达64~128根，部分测试设备超过256根，天线阵列规模提升数十倍 |
| 波束赋形（Beamforming） | 通过精密控制各天线单元的相位和幅度，使无线信号在目标方向叠加增强，在干扰方向相互抵消，形成"笔形波束"精准指向用户 |
| MU-MIMO：多用户空间复用 | 基站同时向多个方向不同用户发送独立数据流，频谱效率理论上与天线数成正比提升，这是5G容量大幅提升的核心机制之一 |
| 产品设计影响            | 5G终端需支持多天线收发，PA/LNA数量增加，天线排布更复杂；这也是5G手机较4G更厚、功耗更高、发热量更大的主要原因 |

**5G网络切片：按需定制的虚拟化专属网络**

| 技术               | 功能说明                                                     |
| ------------------ | ------------------------------------------------------------ |
| 什么是网络切片     | 在同一套5G物理基础设施上，通过SDN和NFV技术虚拟化出多条逻辑专用网络，每条切片拥有独立资源、QoS保障和强隔离性 |
| SA架构的专属特性   | 网络切片是SA独立组网的核心特性，NSA架构无法实现完整端到端切片——这是5G与4G在架构层面最本质的差异之一 |
| 工业互联网典型部署 | 同一工厂园区一套5G基站，生产控制PLC走uRLLC切片（1ms延迟），高清质检摄像头走eMBB切片（高带宽），传感器走mMTC切片（超多连接） |
| to B产品设计视角   | 5G to B（面向企业）场景中，网络切片是5G专网方案的核心卖点；产品设计师需理解切片SLA（服务级别协议）对产品性能的约束和保障 |

海尔COSMOPlat × 5G专网：以三切片同时承载三类工业业务

海尔、富士康、美的等制造商于2020-2022年陆续建设5G SA专网工厂。以某家电工厂为例：园区内200+台AGV机器人通过uRLLC切片（延迟<10ms）实时协同作业；全线1080+台高清摄像头通过eMBB切片传输4K视觉AI检测画面；5000+个温度/震动传感器通过mMTC切片实现预测性维护。三类业务共用同一套5G基础设施。

设计一款医院智能输液监测系统，需同时监测1000个病床患者的滴速、余量、异常报警，优先选5G mMTC还是Wi-Fi？核心判断依据是什么？

优先选择 5G mMTC（海量机器类通信）技术方案。

在通信工程中，这种“超高密度、低功耗、小数据量”的场景正是 5G 三大应用场景之一 mMTC。

1. 核心依据：连接密度与抗干扰能力（Connection Density）
   1000 个病床意味着至少有 1000 个传感器在狭小空间内同时运行。
   5G mMTC：设计指标是每平方公里支持 100 万个连接。由于 5G 使用授权频段，各设备之间有严格的资源调度机制，不会发生信道争抢。对于 1000 个设备，5G 基站处理起来游刃有余，信号极稳。
   Wi-Fi：运行在 2.4GHz/5GHz 公共频带。当 1000 个设备尝试连接时，会面临严重的信道冲突（Collision）。此外，医院内还有大量手机、电脑、医疗平板等 Wi-Fi 设备，会导致严重的同频干扰，造成“异常报警”延迟甚至丢包。

2. 关键瓶颈：移动性与覆盖稳定性（Mobility & Coverage）
   输液系统需要保证患者在病床、走廊、卫生间甚至跨楼层移动时，监测不中断。
   Wi-Fi 的痛点：漫游（Roaming）是 Wi-Fi 的天敌。患者从病房走到走廊，设备需要在不同 AP（路由器）之间切换，切换瞬间极易断线，导致监测数据丢失。
   5G mMTC 的优势：5G 采用蜂窝结构，支持无缝切换。基站覆盖范围大且穿透力强，不会像 Wi-Fi 那样存在大量的“信号死角”。

3. 运维挑战：功耗与维护成本（Low Power Consumption）
   输液监测器通常是电池供电的小型设备。
   5G mMTC (如 NB-IoT)：拥有深度睡眠模式（PSM）和长周期断续接收（eDRX），设备在不发送数据时几乎不耗电，电池寿命可达数年，减少了护士频繁给监测器充电或换电池的工作量。
   Wi-Fi：功耗相对较高。即使是 Wi-Fi 6 的 TWT 技术，其待机功耗也难以维持长时间的监测需求，设备体积和充电频率会成为护士的负担。
